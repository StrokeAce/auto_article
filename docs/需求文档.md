## 1. 项目概述

### 1.1 项目名称
AAP (Automated Article Publisher) - 微信公众号文章自动化发布工具

### 1.2 一句话定位
面向**单人单公众号**的本地CLI工具,将Markdown内容按可定制模板渲染为微信图文兼容HTML,支持**API自动上传草稿箱**与**手动拷贝导出**双路径,用户在后台预览确认后手动发布。

### 1.3 核心边界
- ✅ **做**: 内容解析、模板应用、HTML渲染、本地预览、API上传图片到素材库、API上传文章到草稿箱、手动拷贝导出包(HTML+ZIP+指引)、IP白名单代理(SCF)
- ❌ **不做**: 内容生成(由使用者提供)、自动发布(用户手动在后台点发布)

### 1.4 用户画像
- 个人主体,持有**未认证的个人订阅号**
- 已确认可调用草稿箱API + 素材上传API(均实测成功)
- 已获取AppID和AppSecret
- 家庭宽带网络,IP每天变化(已配置过IP白名单,认可SCF方案)
- 具备一定技术能力(可使用CLI、编辑配置文件)
- 希望排版风格统一且可定制,发布流程尽量自动化
- 倾向双路径:API自动为主,手动拷贝作为降级/预验备选

## 2. 核心目标

| 编号 | 目标 | 衡量标准 |
|------|------|----------|
| G1 | 解析Markdown+Front Matter为结构化内容 | 支持标题/段落/列表/表格/图片/代码块/引用块 |
| G2 | 提供可定制的模板系统 | 配置文件+可视化编辑器双轨 |
| G3 | 高保真本地预览 | 三级预览(纯HTML/微信样式模拟/截图对比) |
| G4 | 双路径发布(API+手动) | API模式:自动上传草稿;手动模式:导出HTML+ZIP+指引,用户仅需手动发布 |
| G5 | 解决动态IP问题 | 通过SCF云函数代理固定出口IP |
| G6 | 架构可扩展 | 为未来启用发布API(全自动)预留接口 |

## 3. 用户场景

### 3.1 典型工作流
```
[作者] 撰写 article.md (含Front Matter + 正文 + 本地图片)
   ↓
[CLI] aap preview article.md          # 本地预览渲染效果
   ↓
[作者] 调整模板或Markdown,热重载预览
   ↓
[CLI] aap publish article.md          # API发布到草稿箱
   ↓
[CLI] 自动执行:
        1. 渲染最终HTML(应用模板,内联CSS)
        2. 上传图片到微信素材库(通过SCF代理) → 获取media_id和URL
        3. 替换HTML中的图片引用为微信图片URL
        4. 调用草稿箱API上传文章(通过SCF代理) → 获取草稿media_id
   ↓
[CLI] 输出: "✅ 已上传到草稿箱,media_id=xxx,请登录后台预览并发布"
   ↓
[作者] 登录 mp.weixin.qq.com → 草稿箱 → 预览 → 发布
```

### 3.2 模板定制流
```
[CLI] aap template edit tech-style    # 启动本地Web编辑器
   ↓
[浏览器] 左侧调参(字号/颜色/间距/图片样式等) 右侧实时预览
   ↓
[CLI] 保存为 ~/.aap/templates/tech-style/{template.yaml, style.css}
   ↓
[作者] 在 article.md 的Front Matter中声明 template: tech-style
```

### 3.3 SCF代理部署流(首次使用)
```
[作者] 腾讯云控制台开通SCF服务,获取SecretId/SecretKey
   ↓
[CLI] aap deploy scf                  # 一键部署SCF代理函数
   ↓
[CLI] 自动: 创建SCF函数 → 配置固定出口IP → 返回SCF URL
   ↓
[作者] 将SCF固定IP加入微信公众号IP白名单
   ↓
[CLI] aap config set scf.url <url>    # 配置本地AAP使用SCF代理
```

### 3.4 手动拷贝导出流(降级/预验)
```
[CLI] aap export article.md           # 生成手动拷贝包(不改API)
   ↓
[CLI] 输出: render_output/<article>/
        ├── article.html             # 可粘贴到微信编辑器的HTML
        ├── images.zip               # 图片打包(按顺序命名)
        ├── manifest.json            # 图片清单(占位media_id为null)
        └── INSTRUCTIONS.txt         # 操作指引
   ↓
[作者] 手动登录 mp.weixin.qq.com
   ↓
[作者] 新建图文 → 粘贴HTML → 上传images.zip中图片 → 替换占位 → 保存草稿
   ↓
[作者] (可选) aap image bind 替换占位为微信URL
```

**适用场景**:
- API/SCF故障时的降级方案
- 发布前想先本地验证HTML兼容性
- 网络不通或临时无SCF可用
- 想手动控制每一步

## 5. 功能需求

### FR-1: 内容输入与解析

**输入**: 单个 `.md` 文件

**Front Matter字段定义** (YAML格式):
```yaml
---
title: 文章标题                    # 必填,微信草稿title字段
author: 作者名                     # 可选,默认取config.author
summary: 文章摘要                  # 可选,微信草稿digest字段,缺省取正文首段
cover: cover.jpg                  # 必填,封面图本地路径(建议900x500,2.35:1)
tags: [技术, 教程]                 # 可选,仅本地分类用
template: tech-style              # 可选,指定模板名,缺省用config.default_template
category: 原创                    # 可选,转载/原创标记
---
```

**正文支持元素**:
- 标题: `#` ~ `######` (映射为微信样式)
- 段落: 普通文本
- 强调: `**粗体**` / `*斜体*` / `~~删除线~~`
- 列表: 有序/无序
- 表格: 标准Markdown表格
- 图片: `![alt](path)` (path为本地相对路径)
- 代码块: ```语言 代码 ```
- 引用块: `> 文本`
- 分割线: `---`
- 链接: `[文本](url)` (微信会转为蓝字)

**解析输出**: 结构化AST + 元数据字典

### FR-2: 模板系统

**模板存储位置**:
- 全局模板: `~/.aap/templates/<name>/`
- 项目模板: `./.aap/templates/<name>/` (优先级更高)
- 内置预设: AAP自带 `tech / essay / minimal` 三套

**模板文件结构**:
```
templates/tech-style/
├── template.yaml    # 排版参数配置
└── style.css        # CSS样式表(微信兼容子集)
```

**template.yaml 字段** (可视化编辑器可调):
```yaml
name: tech-style
description: 科技博客风格,清爽理性

typography:              # 文字样式
  font_family: -apple-system, "PingFang SC", "Microsoft YaHei"
  base_size: 16px
  base_color: "#333333"
  line_height: 1.75
  heading_color: "#1a1a1a"
  heading_sizes: [h1: 24px, h2: 20px, h3: 18px, h4: 16px]
  link_color: "#576b95"  # 微信经典蓝
  code_color: "#c7254e"

spacing:                 # 段落与间距
  paragraph_margin: 16px
  first_line_indent: 0   # 0或2em
  left_right_padding: 16px
  background_color: "#ffffff"

image:                   # 图片样式
  max_width: 100%
  border_radius: 4px
  shadow: false
  caption_color: "#999999"
  caption_size: 14px

table:                   # 表格样式
  border_color: "#e5e5e5"
  header_background: "#f7f7f7"
  zebra_striped: true
  zebra_color: "#fafafa"

code_block:              # 代码块样式
  theme: github          # 内置主题
  background: "#f6f8fa"
  font_size: 14px
  line_numbers: true
```

**模板选择优先级**:
1. Front Matter的`template`字段
2. CLI参数`--template`
3. 项目配置`default_template`
4. 全局配置`default_template`
5. 内置`minimal`

### FR-3: 可视化编辑器

**入口**: `aap template edit [name]`

**行为**:
1. 启动本地FastAPI服务(默认端口7789)
2. 自动打开默认浏览器访问编辑器
3. 加载指定模板(或新建模板)
4. 左侧参数面板 + 右侧实时预览(示例文章)
5. 修改参数即时反映到预览
6. 点击"保存"写入YAML+CSS

**可编辑范围**:
- ✅ 文字样式: 字体/字号/颜色/行高/标题层级
- ✅ 段落与间距: 段间距/首行缩进/左右边距/背景色
- ✅ 图片样式: 圆角/阴影/宽度/对齐/说明文字样式
- ✅ 表格与代码块: 边框/斑马纹/代码主题/行号

**技术实现**:
- 前端: 简单HTML+原生JS(或轻量框架),避免构建复杂度
- 后端: FastAPI提供模板读写+预览渲染API
- 通信: REST + WebSocket(预览热更新)

### FR-4: 本地预览

**入口**: `aap preview <article.md> [--mode <mode>]`

**三种预览模式**:

| 模式 | 命令 | 说明 | 优先级 |
|------|------|------|--------|
| A: 纯HTML | `--mode html` | 最快渲染,标准浏览器样式 | MVP |
| B: 微信样式模拟 | `--mode wechat` (默认) | 嵌入微信内置浏览器内核样式表,字号/颜色/边距贴近真实 | MVP |
| C: 截图对比 | `--mode screenshot` | 用Playwright渲染并截图,与微信端真实截图比对 | P1后续迭代 |

**微信样式模拟要点**:
- 模拟微信内置浏览器(X5内核/Blink)渲染
- 字号映射: 微信字号档位(小/默认/大/超大)
- 颜色: 微信经典配色(链接蓝#576b95等)
- 边距: 微信图文左右各16px内边距
- 字体: 苹方/雅黑回退链

**热重载**:
- 监听Markdown文件变更 → 自动重新渲染
- 监听模板文件变更 → 自动重新渲染
- WebSocket推送更新到浏览器

**预览页面**:
- 顶部: 文章元信息(标题/作者/摘要/封面/模板名)
- 中部: 文章正文渲染区(可切换模式)
- 底部: 元素检查面板(点击元素查看应用的样式)

### FR-5: 图片处理(API自动上传)

**自动扫描**: 解析Markdown中所有`![](path)`本地图片引用

**处理流程**:
1. 扫描所有本地图片引用
2. 调用微信素材上传API(通过SCF代理)逐张上传
3. 获取每张图片的 `media_id` 和微信URL
4. 替换HTML中的图片引用为微信图片URL
5. 生成 `manifest.json` 记录映射关系(便于追溯)

**API调用**:
- 接口: `POST https://api.weixin.qq.com/cgi-bin/material/add_material`
- 参数: `access_token`, `type=image`, multipart文件上传
- 返回: `media_id` 和 `url`

**manifest.json** (记录用):
```json
{
  "article": "article.md",
  "generated_at": "2026-07-05T10:30:00",
  "images": [
    {
      "index": 1,
      "original_path": "images/diagram.png",
      "alt": "架构图",
      "wechat_media_id": "abc123def456",
      "wechat_url": "https://mmbiz.qpic.cn/...",
      "upload_time": "2026-07-05T10:30:05"
    }
  ]
}
```

**降级方案** (API不可用时):
- 生成 `images.zip` + `manifest.json`(占位media_id为null)
- 提供 `aap image bind` 交互命令,用户手动上传后填回media_id
- 自动替换HTML占位为微信URL

**封面图处理**:
- Front Matter的 `cover` 字段单独上传
- 作为草稿的 `thumb_media_id` 参数

### FR-6: 草稿上传(API自动)

**入口**: `aap publish <article.md>`

**完整流程**:
1. 解析Markdown + Front Matter
2. 应用模板渲染为微信兼容HTML(内联CSS)
3. 上传所有正文图片到素材库(FR-5)
4. 上传封面图到素材库
5. 替换HTML中的图片引用为微信URL
6. 调用草稿箱API上传文章
7. 输出结果与操作指引

**草稿箱API调用**:
- 接口: `POST https://api.weixin.qq.com/cgi-bin/draft/add`
- 参数: `access_token`, JSON body:
  ```json
  {
    "articles": [{
      "title": "文章标题",
      "author": "作者",
      "digest": "摘要",
      "content": "<html>...</html>",
      "thumb_media_id": "封面图media_id",
      "need_open_comment": 0,
      "only_fans_can_comment": 0
    }]
  }
  ```
- 返回: `media_id` (草稿ID)

**输出**:
```
✅ 文章已上传到草稿箱

  标题: 文章标题
  作者: 作者名
  草稿media_id: abc123def456
  图片数量: 5张(已上传到素材库)

下一步:
  1. 登录 https://mp.weixin.qq.com
  2. 进入 草稿箱
  3. 找到本文草稿,点击预览
  4. 确认排版无误后,点击"发布"
```

**输出目录** (本地留存): `./render_output/<article-name>/`
- `article.html` - 渲染后的HTML(已替换图片URL)
- `manifest.json` - 图片上传记录
- `publish_log.json` - 发布日志(含media_id、时间戳)

### FR-7: 配置管理

**配置文件层级** (优先级从高到低):
1. CLI参数
2. `./.aap/config.yaml` (项目级)
3. `~/.aap/config.yaml` (全局级)
4. 内置默认值

**config.yaml 示例**:
```yaml
account:
  app_id: "wx_xxxxx"              # 必填,微信公众号AppID
  app_secret: "xxxxxx"            # 必填,微信公众号AppSecret
  nickname: "我的公众号"

scf:
  url: "https://xxx.tencentcloudscf.com/"  # SCF代理URL
  secret: "xxxxxx"                         # SCF访问密钥
  enabled: true                            # 是否启用SCF代理

default_template: minimal

preview:
  port: 7788
  default_mode: wechat
  open_browser: true

publish:
  output_dir: ./render_output
  auto_clipboard: false           # API路径下不需要剪贴板

editor:
  port: 7789
  sample_article: ~/.aap/sample.md

history:
  enabled: true                   # 记录发布历史
  file: ~/.aap/history.jsonl
```

**初始化命令**: `aap config init` 引导生成全局配置

### FR-8: SCF代理模块 (P1必需)

> ⚠️ **当前路径C-草稿版必需此模块**。用于解决家庭宽带IP变动导致的IP白名单问题。

**问题背景**:
- 微信公众号API要求调用方IP在白名单内
- 家庭宽带IP通常每天变化
- 手动更新白名单不可持续

**解决方案**: 部署腾讯云SCF云函数作为API调用代理
- SCF函数拥有固定出口IP
- 将该IP加入微信公众号IP白名单(一次性配置)
- 本地AAP通过HTTPS调用SCF,SCF转发到微信API

**SCF函数功能**:
1. 接收本地AAP的HTTPS请求(含API路径+参数+body)
2. 鉴权(验证SCF访问密钥)
3. 转发请求到微信API
4. 返回响应给本地AAP

**CLI命令**:
- `aap deploy scf` - 一键部署SCF函数到腾讯云
  - 需要: 腾讯云SecretId/SecretKey
  - 自动: 创建SCF函数、配置触发器、返回URL
- `aap config set scf.url <url>` - 配置SCF地址
- `aap config set scf.secret <secret>` - 配置SCF密钥
- `aap scf status` - 检查SCF函数状态
- `aap scf ip` - 查询SCF固定出口IP(用于加入微信白名单)

**API客户端自动路由**:
- 若 `scf.enabled=true`: 所有微信API调用走SCF代理
- 若 `scf.enabled=false`: 直接调用微信API(需本地IP在白名单)

### FR-11: 手动拷贝导出(降级/预验路径)

**入口**: `aap export <article.md>`

**定位**: 与 `aap publish` (API路径) 并行的手动拷贝路径,适用于:
- API/SCF故障降级
- 发布前HTML兼容性预验
- 网络不通或无SCF场景
- 想手动控制每一步

**流程**:
1. 解析Markdown + Front Matter
2. 应用模板渲染为微信兼容HTML(内联CSS)
3. 扫描所有本地图片引用
4. 图片用占位标记 `{{IMG_01}}` 替换(不调用API)
5. 打包图片为 `images.zip`(按顺序命名)
6. 生成 `manifest.json`(media_id为null,待绑定)
7. 生成 `INSTRUCTIONS.txt` 操作指引
8. (可选)自动复制HTML到剪贴板(`--no-clip`关闭)

**输出目录**: `./render_output/<article-name>/`
- `article.html` - 可粘贴到微信编辑器的HTML(图片用占位)
- `images.zip` - 图片打包
- `manifest.json` - 图片清单(占位media_id为null)
- `INSTRUCTIONS.txt` - 操作指引

**INSTRUCTIONS.txt 内容示例**:
```
AAP手动发布指引 - article.md
================================

1. 登录微信公众号后台: https://mp.weixin.qq.com

2. 新建图文消息: 草稿箱 → 新建图文

3. 填写图文信息:
   - 标题: <从Front Matter提取>
   - 作者: <从Front Matter提取>
   - 摘要: <从Front Matter提取>
   - 封面图: 上传 <cover.jpg>

4. 图文正文:
   - 方式A(推荐): 使用"导入"功能,选择 article.html
   - 方式B: 复制 article.html 内容,粘贴到编辑器

5. 图片处理:
   - 解压 images.zip
   - 在微信编辑器中逐张上传图片(按 01.png, 02.png 顺序)
   - 替换HTML中的 {{IMG_01}}, {{IMG_02}} 占位

6. (可选) 执行 aap image bind 替换占位为微信URL

7. 预览确认 → 保存草稿
```

**与 `aap publish` 的关系**:
- `aap publish`: API全流程,自动上传图片+草稿,无需人工介入图片
- `aap export`: 仅生成本地包,图片需手动上传,适合降级或预验
- 两者共享: 内容解析、模板渲染、HTML生成逻辑

- **备路径(手动)**: `aap export` - 生成HTML+ZIP+指引,用户手动粘贴上传

### 决策2: 内容与模板映射用Front Matter

**决策**: 在Markdown文件头部YAML Front Matter中通过 `template: <name>` 字段指定模板。

**优先级链**: Front Matter > CLI参数 > 项目配置 > 全局配置 > 内置默认

**理由**: 一站式声明,文章与模板绑定关系清晰,版本控制友好。

### 决策3: 模板系统双轨制

**决策**: 配置文件(YAML+CSS) + 可视化编辑器并存,底层共享同一配置格式。

**理由**:
- 配置文件: 技术用户直接编辑,可Git版本控制,精细控制
- 可视化编辑器: 非技术用户快速调整,所见即所得
- 两者共享格式,互转无损

### 决策4: 图片处理策略(API自动上传)

**决策**: 调用微信素材上传API自动上传图片,获取media_id和URL,自动替换HTML中的图片引用。

**理由**: 路径C-草稿版下API可用,可实现全自动化,避免手动上传的繁琐。

**降级方案**: 若素材API不可用,退化为ZIP打包+`aap image bind`手动绑定。

### 决策5: IP白名单通过SCF云函数解决

**决策**: 部署腾讯云SCF云函数作为微信API调用代理,固定出口IP。

**理由**:
- 家庭宽带IP变动,无法稳定加入白名单
- SCF有免费额度,个人使用成本几乎为零
- SCF函数出口IP固定,一次配置白名单即可
- `aap deploy scf` 一键部署,降低使用门槛

**技术细节**:
- SCF函数接收本地AAP的HTTPS请求
- 鉴权后转发到微信API
- 返回响应给本地AAP
- 本地AAP通过 `scf.enabled` 配置控制是否走代理

### 决策6: 工具形态为CLI+内嵌本地Web预览

**决策**: CLI为入口,需要可视化/预览时启动本地Web服务,浏览器访问。

**理由**:

- Web预览: 可视化编辑器、实时预览必须用浏览器
- 本地服务: 无需部署,启动即用,安全(不暴露公网)

## 9. 已知限制

1. **个人订阅号无发布API权限** - 仅草稿箱API可用,最终发布需人工在后台点击
2. **API调用受IP白名单限制** - 必须通过SCF代理或手动维护白名单
3. **access_token有调用频次限制** - 个人号每日2000次,需合理缓存(2小时有效期)
4. **微信编辑器HTML兼容性** - 草稿content字段HTML需符合微信兼容子集
   - 已知过滤: `<script>`, `<iframe>`, `<style>`(需内联), `position:absolute`, 部分CSS属性
   - 已知支持: 基础标签, `style`属性内联, `<section>`, `<span>`, 表格, 图片
5. **微信端渲染差异** - 微信内置浏览器(X5/Blink)与标准浏览器有差异,本地预览不能100%还原
6. **SCF依赖腾讯云** - 需要腾讯云账号,免费额度有限(但个人使用足够)
7. **图片格式限制** - 微信素材API支持png/jpeg/jpg/gif,单文件2M内
