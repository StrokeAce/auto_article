## 1. 概述

本文档基于需求文档v0.3,定义AAP的模块设计、接口定义、数据结构与MVP开发计划。

### 1.1 设计原则
- **模块解耦**: 解析/渲染/发布各层独立,可单测
- **双路径共享**: API publish 与手动 export 共享解析渲染逻辑
- **配置驱动**: 模板/账号/SCF配置均YAML化
- **渐进式**: MVP先跑通主链路,再补全降级路径

#### 新增能力（v0.2）

1. **H1-H6 转 P 标签**（`_convert_subheadings_to_paragraphs`）
   - 目的：规避微信编辑器对 h1-h6 内置 font-size 的强制覆盖
   - 实现：正则 `<(h[1-6])...>...</\1>` 替换为 `<p>...</p>`，保留内联样式
   - 执行时机：在 inline_css 之后执行

2. **整图模式章节装饰**（`_replace_chapter_titles_with_images`）
   - 每个 H1 按出现顺序使用对应编号的图片
   - 替换结构：`<section><img><span>标题</span></section>`

3. **单图标模式章节装饰**（`_decorate_chapter_titles`）
   - 所有 H1 共用同一张图标
   - 替换结构：`<section><img><span>标题</span></section>`

4. **三种渲染入口**
   - `render_preview()`：保留原图片 src（本地预览）
   - `render_for_export()`：图片用 `{{IMG_01}}` 占位符
   - `render_for_publish()`：图片用微信 URL

#### CSS 伪元素/伪类支持（v0.2 新增）

`inline_css` 函数新增三个处理分支：

1. **`::before` 伪元素**（`_apply_before_pseudo`）
   - 支持 `content` 属性中的 `counter()` 函数
   - 支持 `decimal-leading-zero` 格式（01、02、03...）
   - 实现：在匹配元素前插入 `<span style="...">CHAPTER 01</span>`
   - 示例：`h1::before { content: "CHAPTER " counter(chapter, decimal-leading-zero); }`

2. **`::after` 伪元素**（`_apply_after_pseudo`）
   - 在匹配元素后插入 `<span>`

3. **`:nth-child(even)` 伪类**（`_apply_nth_child_even`）
   - 支持两种形式：`tr:nth-child(even)` 和 `tr:nth-child(even) td`
   - 用于实现斑马纹表格

class WeChatClient:
    """微信API统一客户端,自动路由SCF代理"""

    def __init__(self, config: AppConfig):
        self.config = config
        self.auth = AuthManager(config)
        self.material = MaterialAPI(self)
        self.draft = DraftAPI(self)
        self.http = httpx.AsyncClient(timeout=30)

    async def request(self, method: str, path: str, **kwargs) -> dict:
        """
        发起API请求
        - 若 scf.enabled: 通过SCF代理
        - 否则: 直连微信API
        """
        if self.config.scf.enabled:
            return await self._request_via_scf(method, path, **kwargs)
        else:
            return await self._request_direct(method, path, **kwargs)

    async def _request_via_scf(self, ...):
        """通过SCF代理调用"""

    async def _request_direct(self, ...):
        """直连微信API"""

#### 目录模式（v0.2 新增）

预览服务支持双模式启动：

1. **单文件模式**：`PreviewServer(md_path, ...)` — md_path 是 .md 文件
2. **目录模式**：`PreviewServer(dir_path, ...)` — md_path 是目录

目录模式路由：
- `GET /`：文章列表页（扫描所有 .md 文件，按修改时间倒序）
- `GET /view/{md_stem}`：渲染指定 md 文件（target="_blank" 新窗口打开）
- `GET /view/{md_stem}/raw`：返回原始渲染 HTML
- `GET /raw`：单文件模式返回原始 HTML
- `GET /image/{path:path}`：本地图片访问
- `GET /icon/{filename}`：模板章节图标（单图标模式）
- `GET /icon/{sub_dir}/{idx}/{filename}`：章节标题整图（整图模式）
- `WS /ws`：WebSocket 热重载（轮询当前 md 文件 mtime）

文章列表页特性：
- 表格展示：标题/模板/路径/修改时间/大小
- 链接 target="_blank"：在新标签页打开
- 工具栏：显示文件名、模板、模式，提供"返回列表/查看原始 HTML"链接

### 4.2 微信API接口

| 功能 | 方法 | 路径 | 参数 |
|------|------|------|------|
| 获取token | GET | /cgi-bin/token | grant_type=client_credential, appid, secret |
| 上传素材 | POST | /cgi-bin/material/add_material | access_token, type=image, file |
| 新建草稿 | POST | /cgi-bin/draft/add | access_token, articles JSON |
| 获取草稿 | POST | /cgi-bin/draft/get | access_token, media_id |
| 删除草稿 | POST | /cgi-bin/draft/delete | access_token, media_id |

### 4.3 SCF代理接口

**请求格式** (本地AAP → SCF):
```json
{
  "secret": "SCF_ACCESS_SECRET",
  "method": "POST",
  "path": "/cgi-bin/draft/add",
  "params": {"access_token": "xxx"},
  "body": {"articles": [...]},
  "body_type": "json"
}
```

**文件上传特殊处理**:
```json
{
  "secret": "SCF_ACCESS_SECRET",
  "method": "POST",
  "path": "/cgi-bin/material/add_material",
  "params": {"access_token": "xxx", "type": "image"},
  "body_type": "multipart",
  "file_name": "01.png",
  "file_content_base64": "..."
}
```

**响应格式** (SCF → 本地AAP):
```json
{
  "status_code": 200,
  "body": {"media_id": "xxx", "url": "..."},
  "error": null
}
```

## 8. 关键技术要点

### 8.1 微信兼容HTML
- **内联CSS**: 微信过滤`<style>`标签,所有样式必须内联到`style`属性
- **标签白名单**: 支持 section/span/p/h1-h6/strong/em/img/table/ul/ol/li/blockquote/pre/code
- **过滤标签**: script/iframe/style/form/input/object/embed
- **图片**: 必须用微信URL(mmbiz.qpic.cn),本地路径不显示

### 8.2 access_token缓存
- 有效期2小时,提前5分钟刷新
- 缓存到`~/.aap/token_cache.json`
- 并发请求用锁防止重复获取

### 8.3 SCF代理安全
- 请求签名: HMAC-SHA256(SCF_SECRET, timestamp + body)
- 时间戳验证: 拒绝5分钟外的请求
- IP白名单: 可选,只允许配置的IP调用SCF

### 8.4 图片上传优化
- 大图压缩(可选,Pillow)
- 并发上传(限制3并发)
- 失败重试(3次,指数退避)
